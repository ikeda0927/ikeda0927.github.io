新規プロジェクトの作成


何はともあれ新しいプロジェクトを作ってみます。

まず、AndroidStudioを起動して"Start a new Android Studio project"をクリックします。

もし、もう既に何かしらのプロジェクトを作っている方は"File"→"New"→"New project..."をクリックしてください。



次に、作成するAndroid アプリの名前とかを決めます。

Application nameは作成するアプリの名前で、Company domainは作成者を識別する感じのものです。

Project locationは今から作成するプロジェクトをどこに置くかを決めるものです。

今回は名前を"TestApplication1"とし、Company domainを"example.example.com"(エラーさえ出なければ何でも良いです。)とします。

また、今回はJavaのみを使用するので"Include C++ support"と"Include Kotolin support"にチェックをしません。

そこまでいったら、"Next"をクリックし、次に進みます。



ここでは対応させる最低限のAPIレベルを決めます。

簡単にいうと、どのくらい古いAndroidまでこれから作るアプリを対応させるかというものです。

2018年12月現在、"Ice Cream Sandwich"というコード名のAndroidが公式のサポートから外れるということなので、その次の"Jelly Bean"(API 16)を最低限のAPIレベルとします。

選択したら、"Next"をクリックし次に進みます。



すると、以下のような今までとはちょっと異なる画面がでてきます。

ここでは"Empty Activity"を選択します。

他の画面が気になる方は試してみるといいと思いますが、RPGを作るんであれば、"Empty Activity"をおすすめします！



"Empty Activity"を選択したら、以下のような画面が出てくると思うのですが、ここではActivityの名前を決めます。

ここで、Activityとは何かということになるんですが、簡単にいいますとユーザが操作できる画面を提供するアプリケーションの構成要素です。

Activityについて詳しく知りたい方はこちらをご覧ください。

まぁ、Activityの名前についても特にこだわりが無い場合はデフォルトの名前でいいと思います。

設定が終わったら、Finishボタンを押します。

Finishボタンを押すと、少々時間がかかりますがプロジェクトの作成が始まります。



プロジェクトが作成されると以下のような画面が現れます。

上部に、activity_main.xmlと、MainActivity.javaというのがタブになっているのが分かると思いますが、ここで編集したいファイルを選択することが出来ます。

そして、現在選択されている(明るく見えるタブ)"MainActivity.java"というのが、先ほど名前を決めたActivityのソースコードです。

基本的にはその下の部分を編集したりします。



ちなみに、"activity_main.xml"のタブを選択すると、以下のような画面が出てきます。

もしかしたら、xml形式の表示がされているかもしれないですが、そのときは下の方にある"Design"っていうのを押してください。そうすればこんな感じの画面になります。

とはいえ、今回xmlを使わないのでこの画面はあまり関係ないです。



エミュレータの設定

次に、Androidのエミュレータの設定をします。

Androidのエミュレータは、これから作るアプリをパソコン上の仮想的な端末で実行するためのものです。

上の図の青で囲まれた部分をクリックすると以下のような画面が出てエミュレータの起動or作成が出来ます。

今回は下の赤線で引かれた"Create Virtual Device"をクリックして新しいエミュレータを作成します。



すると、以下のような画面になります。

ここでは"Pixel 2"を選択します。

選択したら、"Next"をクリックします。



ここでは、API28を選択します。もしかしたら、Release NameがAPI 28 ではなくて"Pi"になってるかもしれませんがそれはそれで大丈夫です。

また、青字で"Download"と書かれている場合は、クリックしてダウンロードしてください。(通新環境にもよりますが結構時間がかかるかもしれません。)

選択したら、"Next"をクリックして次に進みます。



ここでは何も変えずに"Next"をクリックして次に進みます。



すると、先ほど作成した"Pixel 2 API 28"というのが追加されていると思います。

そこで、右の方にある三角形(再生ボタンみたいなやつ)を押すとエミュレータが起動できます。



起動したエミュレータはこんな感じです。

このエミュレータで作成したアプリケーションのテストを行なっていきます。



エミュレータを起動した状態で、下の図の緑色の三角形をクリックすると、



"Select Deployment Target"という画面が出ます。

この画面の"Connected Devices"という欄に現在起動しているエミュレータが表示されます。(実機を接続している場合はそれもここに表示されます。)

これをクリックすることで実行できるのでクリックします。

また、この画面の"Available Virtual Devices"という欄にあるエミュレータをクリックすることで、起動と実行を同時に行えます。



以下が実行されたアプリケーションです。

もしかしたら表示がちょっと違うかもしれませんが、"Hello world!"っていう文字が出ていると思います。

これはAndroidのプロジェクトを作成したときにも最初から入っていたもので、"activity_main.xml"に書かれています。
